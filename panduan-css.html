CSS (Cascading Style Sheets) ngatur tampilan HTML — warna, ukuran, layout, animasi. Kalau HTML itu kerangka bangunan, CSS itu cat, furnitur, dan dekorasinya.

1

Pengenalan & Cara Pakai CSS

Ada tiga cara menulis CSS: langsung di elemen (inline), di dalam tag <style> (internal), atau di file terpisah yang dihubungkan lewat <link> (external). Cara terbaik untuk project sungguhan adalah external — CSS dan HTML dipisah.

style.css
/* Ini komentar CSS */

/* Anatomi CSS: selector { properti: nilai; } */
h1 {
  color: #818cf8;
  font-size: 32px;
  font-weight: bold;
}

p {
  color: #475569;
  font-size: 15px;
  line-height: 1.7;
}

Judul Halaman

Ini paragraf yang sudah di-style pakai CSS. Warnanya lebih soft dan jarak barisnya lebih nyaman dibaca.

selectorBagian yang menentukan elemen mana yang kena style. Bisa nama tag (h1, p), class (.tombol), atau ID (#header).
property: value;Satu aturan style. Properti adalah apa yang diubah (warna, ukuran, dll), nilai adalah isinya. Jangan lupa titik koma di akhir setiap deklarasi!
<link rel="stylesheet" href="style.css">Cara menghubungkan file CSS external ke HTML. Taruh di dalam <head>. Satu file CSS bisa dipakai oleh banyak halaman HTML.
CaraContohKapan Dipakai
Inlinestyle="color:red"Override darurat, jangan kebiasaan
Internal<style>...</style>Halaman tunggal, belajar/eksperimen
External<link href="style.css">Project sungguhan — selalu pakai ini
2

Selector — "Pilih Elemen yang Mau Di-style"

Selector menentukan elemen HTML mana yang kena aturan CSS. Ada banyak jenis selector — dari yang paling basic (nama tag) sampai yang spesifik (kombinasi class + state).

selector.css
/* 1. Tag selector — semua elemen p */
p { color: #333; }

/* 2. Class selector — elemen dengan class="tombol" */
.tombol {
  background: #818cf8;
  color: white;
  padding: 8px 16px;
  border-radius: 6px;
}

/* 3. ID selector — elemen dengan id="judul" */
#judul { font-size: 28px; }

/* 4. Kombinasi — p di dalam .kartu */
.kartu p { color: #555; }

/* 5. Multiple selector — h1 DAN h2 dapat style yang sama */
h1, h2 { font-family: sans-serif; }

/* 6. Universal selector — semua elemen */
* { box-sizing: border-box; }
.classPakai titik di depan. Satu class bisa dipakai di banyak elemen, dan satu elemen bisa punya banyak class: class="tombol merah besar". Ini yang paling sering dipakai.
#idPakai tanda pagar. ID harusnya unik — satu ID hanya ada satu di satu halaman. Lebih spesifik dari class, tapi lebih jarang dipakai untuk styling.
.kartu pDescendant selector: pilih semua p yang ada di dalam elemen ber-class kartu. Berguna untuk styling yang kontekstual tanpa tambah class baru.
Specificity (Kekhususan): CSS punya aturan prioritas. Kalau dua rule bentrok, yang lebih spesifik menang. Urutannya: inline style > ID > class > tag. Makanya inline style selalu override semuanya.

Box Model — "Setiap Elemen itu Kotak"

Di CSS, setiap elemen HTML adalah sebuah kotak dengan 4 lapisan: content (isi), padding (jarak dalam), border (garis tepi), dan margin (jarak luar). Pahami ini, dan lo bisa kontrol layout dengan tepat.

box.css
.kotak {
  /* Ukuran konten */
  width: 200px;
  height: 80px;

  /* Jarak dalam (konten ke border) */
  padding: 16px 24px;

  /* Garis tepi */
  border: 2px solid #818cf8;
  border-radius: 8px;

  /* Jarak luar (dari elemen lain) */
  margin: 20px auto;

  background: #f0f0ff;
}

padding: 16px 24pxShorthand: nilai pertama untuk atas-bawah, kedua untuk kiri-kanan. Bisa juga 4 nilai: padding: atas kanan bawah kiri (searah jarum jam).
margin: 20px autoauto untuk kiri-kanan membuat elemen block menjadi terpusat secara horizontal. Trik klasik untuk centering elemen yang punya width.
box-sizing: border-boxSecara default, width hanya menghitung konten — padding dan border ditambahkan di luar. Pakai box-sizing: border-box agar padding dan border masuk ke dalam ukuran width. Hampir semua developer modern selalu set ini ke semua elemen (* { box-sizing: border-box; }).

Teks & Warna — "Yang Paling Sering Diutak-atik"

Styling teks dan warna adalah hal pertama yang lo otak-atik di CSS. Propertinya banyak tapi yang penting cuma beberapa.

teks.css
.judul {
  font-family: 'Segoe UI', sans-serif;
  font-size: 24px;
  font-weight: 700;
  color: #1e293b;
  letter-spacing: -0.5px;
}

.teks-body {
  font-size: 15px;
  line-height: 1.7;
  color: #475569;
  text-align: justify;
}

.highlight {
  background: #fef9c3;     /* hex */
  color: rgb(30, 41, 59); /* rgb */
  border-left: 4px solid rgba(251,191,36,0.8);
}
Live Preview
Judul Artikel
Ini contoh teks body yang lebih nyaman dibaca karena line-height 1.7 dan warna yang tidak terlalu hitam.
Ini teks dengan highlight kuning.
font-weight: 700Tebal teks. Bisa angka 100–900 (100=tipis, 400=normal, 700=bold, 900=hitam) atau kata kunci normal / bold. Tidak semua font punya semua ketebalan.
line-height: 1.7Jarak antar baris. Tanpa satuan berarti dikali font-size — 1.7 artinya jarak baris = 1.7× ukuran font. Untuk teks artikel, 1.6–1.8 paling nyaman dibaca.
rgba(r, g, b, a)Seperti rgb tapi ada channel alpha (transparansi) 0–1. rgba(0,0,0,0.5) = hitam 50% transparan. Berguna untuk overlay, shadow, border semi-transparan.

Display & Posisi — "Elemen Mau Ditaruh di Mana?"

Properti display menentukan bagaimana elemen ditampilkan di halaman. Ini salah satu properti CSS yang paling penting untuk dipahami karena langsung mempengaruhi layout.

display.css
/* block: ambil seluruh lebar, mulai baris baru */
div, p, h1 { display: block; }

/* inline: ikut alur teks, tidak bisa set width/height */
span, a, strong { display: inline; }

/* inline-block: ikut alur teks, TAPI bisa set width/height */
.badge {
  display: inline-block;
  padding: 4px 12px;
  background: #818cf8;
  border-radius: 99px;
}

/* none: sembunyikan elemen sepenuhnya */
.tersembunyi { display: none; }

/* position: relative + absolute untuk tumpuk elemen */
.parent { position: relative; }
.badge-pojok {
  position: absolute;
  top: 8px;
  right: 8px;
}
display: blockElemen mulai dari baris baru dan memenuhi seluruh lebar container. div, p, h1–h6 defaultnya block. Bisa set width dan height.
display: inline-blockKombinasi inline (ikut alur teks, tidak mulai baris baru) tapi tetap bisa dikasih width, height, padding, dan margin atas-bawah. Cocok untuk tombol, badge, chip.
position: absoluteElemen dilepas dari alur normal dan diposisikan relatif terhadap ancestor terdekat yang punya position: relative. Kalau tidak ada, relatif ke halaman. Dipakai untuk tooltip, badge notifikasi, dropdown.

Flexbox — "Layout Satu Arah yang Fleksibel"

Flexbox adalah sistem layout CSS untuk mengatur elemen dalam satu arah — horizontal atau vertikal. Sangat powerful untuk navbar, card row, centering elemen, dan layout sederhana.

flexbox.css
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 12px 24px;
  background: #1e293b;
}

.kartu-row {
  display: flex;
  gap: 16px;
  flex-wrap: wrap;
}

.kartu {
  flex: 1;        /* ambil ruang sama rata */
  min-width: 200px;
}

/* Centering sempurna */
.tengah {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 200px;
}

justify-contentMengatur posisi elemen di sumbu utama (horizontal kalau row, vertikal kalau column). Nilai: flex-start, flex-end, center, space-between, space-around.
align-itemsMengatur posisi elemen di sumbu silang (vertikal kalau row). center = elemen rata tengah secara vertikal. Ini solusi centering vertikal yang paling mudah.
flex: 1Elemen mengambil ruang yang tersedia secara proporsional. Kalau semua elemen punya flex:1, mereka berbagi ruang sama rata. Kalau satu punya flex:2, dia dapat 2× lebih lebar.
gapJarak antar elemen flex tanpa perlu margin manual. Lebih bersih dari margin-right karena tidak ada sisa di elemen terakhir.

Grid — "Layout Dua Dimensi"

Kalau Flexbox untuk satu arah, CSS Grid untuk dua arah sekaligus — baris dan kolom. Sempurna untuk layout halaman, galeri foto, dashboard.

grid.css
/* Grid 3 kolom sama lebar */
.galeri {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

/* Layout sidebar + konten */
.layout {
  display: grid;
  grid-template-columns: 240px 1fr;
  gap: 24px;
}

/* Auto-fill: otomatis sesuai lebar container */
.kartu-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}

240px 1frKolom pertama lebar tetap 240px (sidebar), kolom kedua ambil sisa ruang yang ada (konten). Pola klasik untuk layout dua kolom.

Pseudo-class & Pseudo-element

Pseudo-class memilih elemen berdasarkan state (kondisi) — seperti saat di-hover, diklik, atau anak pertama. Pseudo-element membuat/memilih bagian virtual dari elemen.

pseudo.css
/* :hover — saat mouse di atas elemen */
.tombol {
  background: #818cf8;
  transition: background 0.2s;
}
.tombol:hover {
  background: #6366f1;
}

/* :nth-child — baris ganjil tabel */
tr:nth-child(odd) {
  background: #f8fafc;
}

/* :first-child dan :last-child */
li:first-child { font-weight: bold; }

/* ::before dan ::after — konten virtual */
.label::before {
  content: "★ ";
  color: #fbbf24;
}

★ Ini teks dengan ::before bintang
:hoverState saat kursor mouse ada di atas elemen. Sering dikombinasi dengan transition biar perubahan stylenya smooth, tidak langsung snap.
:nth-child(odd/even)Pilih elemen berdasarkan posisinya di parent. odd=ganjil, even=genap. Bisa juga pakai formula: :nth-child(3n) = setiap kelipatan 3.
::before / ::afterMembuat elemen virtual sebelum/sesudah konten. Wajib ada properti content (boleh kosong: content: ""). Berguna untuk dekorasi, ikon, atau efek tanpa tambah HTML.

CSS Variable — "Simpan Nilai yang Sering Dipakai"

CSS Variable (custom property) memungkinkan lo menyimpan nilai yang sering dipakai — warna tema, ukuran font, jarak — lalu memanggilnya di mana saja. Kalau mau ganti warna utama, cukup ubah di satu tempat.

variabel.css
/* Deklarasi di :root = tersedia global */
:root {
  --warna-utama: #818cf8;
  --warna-bg: #f8fafc;
  --radius: 8px;
  --font-besar: 18px;
}

/* Pakai dengan var() */
.tombol {
  background: var(--warna-utama);
  border-radius: var(--radius);
  font-size: var(--font-besar);
}

.kartu {
  background: var(--warna-bg);
  border-radius: var(--radius);
}
Live Preview
Kartu — pakai --radius dan --warna-bg yang sama.
Ganti --warna-utama di :root → tombol langsung ikut berubah.
--nama-variabelNama variabel CSS selalu diawali dua strip (--). Huruf besar-kecil berpengaruh: --Warna ≠ --warna. Konvensi umum pakai huruf kecil dengan strip.
var(--nama, fallback)Pakai variabel dengan var(). Bisa tambah fallback: var(--warna-utama, blue) artinya kalau variabel tidak ada, pakai biru. Berguna untuk library atau komponen yang bisa dikustomisasi.

Transisi & Animasi — "Bikin Halaman Hidup"

transition bikin perubahan style jadi smooth — misal hover yang tidak langsung snap. animation lebih powerful: bisa loop, bisa complex multi-step, tanpa perlu trigger dari user.

animasi.css
/* Transisi: properti durasi easing */
.tombol {
  background: #818cf8;
  transform: scale(1);
  transition: all 0.2s ease;
}
.tombol:hover {
  background: #6366f1;
  transform: scale(1.05);
}

/* Animasi: definisi @keyframes */
@keyframes fadeUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.muncul {
  animation: fadeUp 0.5s ease forwards;
}
Live Preview
Elemen muncul dari bawah ✨
transition: all 0.2s easeFormat: properti durasi easing delay. all berarti semua properti yang berubah ikut transisi. Easing: ease (default), linear, ease-in, ease-out, ease-in-out.
transform: scale / translateYtransform tidak mempengaruhi layout — elemen lain tidak bergerak. Lebih performa daripada mengubah width/margin karena diproses di GPU.
@keyframesDefinisi animasi multi-step. Bisa pakai from/to untuk 2 langkah, atau persentase: 0% { } 50% { } 100% { } untuk animasi yang lebih kompleks.

Responsive Design — "Tampilannya Rapi di Semua Layar"

Responsive design artinya layout menyesuaikan diri dengan ukuran layar. Caranya: media query untuk override CSS di ukuran tertentu, plus unit relatif seperti %, vw, dan clamp().

responsive.css
/* Default: mobile first */
.grid {
  display: grid;
  grid-template-columns: 1fr;    /* mobile: 1 kolom */
  gap: 16px;
}

/* Tablet ke atas */
@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

/* clamp(min, ideal, max) — font yang fluid */
h1 {
  font-size: clamp(24px, 5vw, 52px);
}
@media (min-width: 768px)Aturan CSS di dalam blok ini hanya aktif kalau lebar layar ≥ 768px. Ini pendekatan mobile-first — tulis default untuk mobile dulu, override untuk layar lebih besar.
vw / vhUnit relatif terhadap viewport. 1vw = 1% lebar layar, 1vh = 1% tinggi layar. 100vh = tinggi penuh layar — berguna untuk hero section atau full-screen layout.
clamp(min, ideal, max)Font/ukuran yang fleksibel: tidak kurang dari min, tidak lebih dari max, tapi ideally nilai tengah. clamp(24px, 5vw, 52px) = di mobile 24px, di desktop besar 52px, di tengah fleksibel.
Jangan lupa tag ini di <head>: <meta name="viewport" content="width=device-width, initial-scale=1.0">. Tanpa ini, media query tidak berfungsi di mobile!

Tips & Trik CSS

Beberapa shortcut dan kebiasaan baik yang bikin CSS lo lebih rapi dan efisien.

tips.css
/* Reset ringan — selalu taruh di awal */
*, *::before, *::after {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

/* Centering sempurna dengan flex */
.pusat {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Teks tidak meluber */
.teks-ellipsis {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

/* Gambar responsif */
img {
  max-width: 100%;
  height: auto;
}

/* Smooth scroll bawaan */
html {
  scroll-behavior: smooth;
}
TrikFungsi
outline: noneHilangkan outline default browser (hati-hati aksesibilitas!)
cursor: pointerUbah kursor jadi tangan — kasih ke elemen yang bisa diklik
user-select: noneCegah teks bisa diseleksi — berguna untuk tombol/label
overflow: hiddenSembunyikan konten yang meluber dari container
z-indexAtur tumpukan elemen — yang lebih besar di depan
opacity: 0.5Transparansi elemen 0 (invisible) sampai 1 (full)
Pakai DevTools browser (F12 → tab Elements → Styles) untuk eksperimen CSS langsung di browser. Lo bisa edit nilai properti real-time tanpa ubah file — super berguna buat debugging layout.
