Interaktif & Hidup

JavaScript adalah satu-satunya bahasa pemrograman yang langsung dimengerti browser. Kalau HTML itu struktur dan CSS itu tampilan, JS itu yang bikin tombol bereaksi, data berubah, dan halaman bergerak.

1

Pengenalan JavaScript

JavaScript dijalankan langsung di browser — tidak perlu compile, tidak perlu install. Cukup tulis di tag <script> atau file .js terpisah, langsung jalan. Buka DevTools (F12) → Console untuk eksperimen langsung.

script.js
// Ini komentar satu baris

/* Ini komentar
   multi baris */

// Tampilkan di console browser
console.log("Halo dari JavaScript!");

// Tampilkan popup
alert("Selamat datang!");

// Tanya user
let nama = prompt("Siapa namamu?");
console.log("Halo, " + nama);

2

Variabel & Tipe Data

Di JavaScript modern, pakai let (bisa diubah) dan const (tidak bisa diubah). Hindari var yang lama karena punya perilaku aneh dengan scope.

variabel.js
// let — bisa diubah nilainya
let umur = 20;
umur = 21;           // ok

// const — tidak bisa diubah (gunakan untuk nilai tetap)
const nama = "Budi";
// nama = "Ani";   → Error!

// Tipe data otomatis (dynamic typing)
let teks    = "Halo";       // String
let angka   = 42;          // Number
let desimal = 3.14;        // Number (sama)
let benar   = true;        // Boolean
let kosong  = null;        // Null (sengaja kosong)
let belumAda;             // Undefined (belum diisi)

// Cek tipe dengan typeof
console.log(typeof teks);   // "string"
console.log(typeof angka);  // "number"

// Template literal — string dengan variabel di dalamnya
console.log(`Halo, ${nama}! Kamu ${umur} tahun.`);
Output
string number Halo, Budi! Kamu 21 tahun.
constGunakan const sebagai default — kalau nilainya tidak perlu berubah. Pakai let hanya kalau memang akan diubah. Ini praktik modern yang dianjurkan.
`template ${literal}`Pakai backtick (`) bukan tanda kutip. Variabel atau ekspresi bisa dimasukkan langsung dengan ${...}. Jauh lebih rapi dari penggabungan string dengan +.
typeofOperator untuk mengecek tipe data saat runtime. Berguna untuk debug atau validasi input. Hasilnya selalu string: "string", "number", "boolean", "undefined", "object", "function".
3

Operator

Operator JS mirip dengan bahasa lain — aritmatika, perbandingan, logika. Yang perlu diperhatikan: JS punya dua jenis kesamaan, dan keduanya sangat berbeda perilakunya.

operator.js
// Aritmatika
console.log(10 + 3);   // 13
console.log(10 % 3);   // 1  (modulus/sisa bagi)
console.log(2 ** 8);   // 256 (pangkat)

// == vs === (PENTING!)
console.log(5 ==  "5");  // true  (bandingkan nilai, abaikan tipe)
console.log(5 === "5");  // false (bandingkan nilai DAN tipe) ✓

// Logika
console.log(true && false);  // false (AND)
console.log(true || false);  // true  (OR)
console.log(!true);           // false (NOT)

// Nullish coalescing — fallback kalau null/undefined
let input = null;
console.log(input ?? "default");  // "default"
=== (strict equality)Selalu pakai === bukan ==. Yang dua sama (==) melakukan type coercion — "0" == false hasilnya true, yang sering bikin bug aneh. === tidak melakukan konversi tipe.
?? (nullish coalescing)Mengembalikan nilai kanan kalau nilai kiri adalah null atau undefined. Berbeda dengan || yang juga trigger untuk 0, "", false. Berguna untuk nilai default.
** (pangkat)Operator eksponensial ES2016. 2 ** 10 = 1024. Alternatif: Math.pow(2, 10).
4

if / else & switch

Struktur kondisi JS sama seperti C — if/else if/else dan switch. Plus ada ternary operator untuk kondisi singkat satu baris.

kondisi.js
let nilai = 78;

// if / else if / else
if (nilai >= 85) {
  console.log("A");
} else if (nilai >= 70) {
  console.log("B");   // ini yang jalan
} else {
  console.log("C");
}

// Ternary — kondisi ? jalanKalauBenar : jalanKalauSalah
let status = nilai >= 60 ? "Lulus" : "Tidak Lulus";
console.log(status);  // "Lulus"

// switch
let hari = "Senin";
switch (hari) {
  case "Sabtu":
  case "Minggu":
    console.log("Libur!");
    break;
  default:
    console.log("Hari kerja");
}
Live Preview
Klik Cek untuk lihat hasil
kondisi ? a : bTernary operator: versi singkat if/else untuk satu ekspresi. Bagus untuk assignment singkat, tapi jangan nested ternary — bikin kode susah dibaca.
case "Sabtu": case "Minggu":Dua case berturut-turut tanpa break di antara = keduanya menjalankan blok yang sama (fall-through). Trik untuk handle beberapa kondisi dengan aksi sama.
5

Loop — "Ulang Terus"

JavaScript punya beberapa jenis loop. for dan while mirip C, tapi ada tambahan for...of yang lebih modern dan mudah untuk iterasi array.

loop.js
// for — seperti C
for (let i = 0; i < 3; i++) {
  console.log(`Iterasi ke-${i}`);
}

// while
let n = 1;
while (n <= 5) {
  console.log(n);
  n++;
}

// for...of — iterasi array (lebih modern)
const buah = ["apel", "mangga", "jeruk"];
for (const item of buah) {
  console.log(item);
}

// for...in — iterasi key object
const orang = { nama: "Budi", umur: 20 };
for (const key in orang) {
  console.log(`${key}: ${orang[key]}`);
}
Output
Iterasi ke-0 Iterasi ke-1 Iterasi ke-2 apel mangga jeruk nama: Budi umur: 20
for...ofIterasi nilai dari array (atau iterable lain). Lebih bersih dari for biasa karena tidak perlu index. Gunakan ini untuk array, tinggalkan for biasa untuk kasus yang butuh index.
for...inIterasi key (nama properti) dari object. Untuk ambil nilainya: orang[key]. Jangan pakai for...in di array — hasilnya bisa tidak terduga.
6

Fungsi — "Kode yang Bisa Dipanggil Ulang"

JavaScript punya tiga cara menulis fungsi. Yang modern dan sering dipakai: arrow function — lebih singkat dan punya perilaku this yang lebih predictable.

fungsi.js
// 1. Function declaration (bisa dipanggil sebelum dideklarasi)
function sapa(nama) {
  return `Halo, ${nama}!`;
}

// 2. Arrow function (modern, ringkas)
const tambah = (a, b) => a + b;

// Arrow function dengan badan fungsi
const gradeNilai = (nilai) => {
  if (nilai >= 85) return "A";
  if (nilai >= 70) return "B";
  return "C";
};

// Default parameter
const beriSalam = (nama = "teman") => `Hai, ${nama}!`;

console.log(sapa("Ani"));        // "Halo, Ani!"
console.log(tambah(3, 4));       // 7
console.log(gradeNilai(78));     // "B"
console.log(beriSalam());        // "Hai, teman!"
(a, b) => a + bArrow function satu baris: kalau hanya satu ekspresi, tidak perlu kurung kurawal dan return — hasilnya otomatis di-return. Kalau butuh banyak baris, pakai kurung kurawal dan tulis return manual.
parameter = "default"Default parameter: kalau argument tidak dikirim saat fungsi dipanggil, pakai nilai default. Lebih bersih dari cek manual if (nama === undefined) nama = "teman".

Array — "Daftar Data"

Array JS lebih fleksibel dari array C — bisa campuran tipe, bisa tumbuh/menyusut dinamis, dan punya banyak method built-in yang powerful.

array.js
const nilai = [85, 70, 92, 78];

// Akses dan panjang
console.log(nilai[0]);           // 85
console.log(nilai.length);       // 4

// Tambah / hapus
nilai.push(65);                  // tambah di akhir
nilai.pop();                    // hapus dari akhir

// map — ubah setiap elemen, hasilnya array baru
const doubled = nilai.map(n => n * 2);

// filter — ambil elemen yang memenuhi syarat
const lulus = nilai.filter(n => n >= 75);

// find — cari elemen pertama yang cocok
const pertama = nilai.find(n => n >= 90);

// reduce — akumulasi jadi satu nilai
const total = nilai.reduce((acc, n) => acc + n, 0);

console.log(lulus);    // [85, 92, 78]
console.log(pertama);  // 92
console.log(total);    // 325
.map()Mengubah setiap elemen dengan fungsi yang diberikan, menghasilkan array baru dengan ukuran sama. Array asli tidak berubah. Contoh klasik: konversi data, format tampilan.
.filter()Mengambil elemen yang fungsinya mengembalikan true. Hasilnya array baru yang lebih pendek atau sama panjang. Array asli tidak berubah.
.reduce(callback, initialValue)acc adalah akumulator — nilai yang terus dikumpulkan. 0 adalah nilai awalnya. Berguna untuk jumlah, rata-rata, atau transformasi array ke satu nilai.
8

Object — "Data dengan Nama Properti"

Object adalah kumpulan pasangan key-value. Mirip struct di C, tapi lebih fleksibel — propertinya bisa ditambah, dihapus, bahkan berisi fungsi.

object.js
// Buat object
const mahasiswa = {
  nama:    "Budi",
  nim:     "12345",
  ipk:     3.75,
  sapa() {                   // method (fungsi dalam object)
    return `Saya ${this.nama}`;
  }
};

// Akses properti
console.log(mahasiswa.nama);      // "Budi"
console.log(mahasiswa["ipk"]);   // 3.75 (bracket notation)
console.log(mahasiswa.sapa());    // "Saya Budi"

// Destructuring — ambil properti ke variabel langsung
const { nama, ipk } = mahasiswa;
console.log(nama, ipk);          // "Budi" 3.75

// Spread — copy / gabung object
const updated = { ...mahasiswa, ipk: 3.9 };
thisDi dalam method object, this merujuk ke object itu sendiri. this.nama = akses properti nama dari object yang memiliki method ini. Jangan pakai arrow function untuk method karena this-nya berbeda.
const { nama, ipk } = objDestructuring: ekstrak beberapa properti sekaligus ke variabel terpisah. Jauh lebih ringkas dari const nama = obj.nama; const ipk = obj.ipk;.
{ ...obj, key: newVal }Spread operator: salin semua properti object lama, lalu override properti tertentu. Hasilnya object baru — object asli tidak berubah. Pola ini sangat umum di React.

DOM — "Manipulasi HTML dengan JavaScript"

DOM (Document Object Model) adalah representasi HTML sebagai objek yang bisa dimanipulasi JS. Lewat DOM, lo bisa ubah teks, warna, tambah/hapus elemen — semua secara dinamis tanpa refresh halaman.

dom.js
// Pilih elemen
const judul = document.getElementById("judul");
const semua = document.querySelectorAll(".item");
const satu  = document.querySelector(".item");

// Ubah konten dan style
judul.textContent = "Judul Baru";
judul.style.color = "#fbbf24";

// Toggle class
judul.classList.add("aktif");
judul.classList.remove("aktif");
judul.classList.toggle("aktif");

// Buat elemen baru
const item = document.createElement("li");
item.textContent = "Item baru";
document.querySelector("ul").appendChild(item);
Live Preview
Judul Asli
  • Item awal
querySelectorPilih satu elemen dengan CSS selector. querySelector(".kartu") = elemen pertama ber-class kartu. querySelectorAll mengembalikan semua yang cocok sebagai NodeList.
classList.toggle()Tambah class kalau belum ada, hapus kalau sudah ada. Sangat berguna untuk tombol on/off, show/hide menu, dark mode toggle.
textContent vs innerHTMLtextContent lebih aman — hanya mengubah teks, tag HTML tidak diproses. innerHTML memproses HTML di dalamnya, tapi berbahaya kalau isinya dari input user (bisa XSS injection).

Event — "Reaksi terhadap Aksi User"

Event adalah aksi yang dilakukan user — klik, ketik, hover, scroll. JavaScript "mendengarkan" event ini dengan addEventListener lalu menjalankan fungsi saat event terjadi.

event.js
const tombol = document.querySelector("#tombol");
const input  = document.querySelector("#inputnya");

// Klik
tombol.addEventListener("click", () => {
  console.log("Tombol diklik!");
});

// Ketik di input — event.target.value = nilai input
input.addEventListener("input", (event) => {
  console.log(event.target.value);
});

// Submit form — cegah reload halaman
document.querySelector("form").addEventListener("submit", (e) => {
  e.preventDefault();  // PENTING: cegah reload
  console.log("Form dikirim!");
});

// Event lain yang umum
// "mouseover"  — hover masuk
// "mouseout"   — hover keluar
// "keydown"    — tombol keyboard ditekan
// "change"     — nilai berubah (select, checkbox)
// "DOMContentLoaded" — HTML selesai dimuat

addEventListener(event, callback)Lebih baik dari attribute HTML (onclick="...") karena bisa pasang beberapa listener di satu elemen, dan kode JS terpisah dari HTML.
event.target.valueevent.target adalah elemen yang memicu event. .value untuk ambil nilai input/select/textarea. .checked untuk checkbox.
e.preventDefault()Cegah perilaku default browser. Untuk form: cegah reload halaman. Untuk link: cegah navigasi. Untuk drag: cegah drop default. Sangat sering dipakai di form handling.

Fetch API — "Ambil Data dari Internet"

Fetch API dipakai untuk mengambil data dari server atau API tanpa reload halaman. Hasilnya asinkron — JS tidak berhenti menunggu, tapi lanjut dan proses hasilnya kalau sudah siap.

fetch.js
// async/await — cara modern baca fetch
async function ambilData() {
  try {
    const response = await fetch(
      "https://jsonplaceholder.typicode.com/users/1"
    );

    const data = await response.json();

    console.log(data.name);  // "Leanne Graham"
    console.log(data.email); // "Sincere@april.biz"

  } catch (error) {
    console.error("Gagal:", error);
  }
}

async / awaitasync di depan fungsi artinya fungsi ini asinkron. await dipakai di dalam fungsi async untuk "tunggu" sampai Promise selesai sebelum lanjut. Kode jadi terlihat synchronous padahal async.
response.json()Parse body response sebagai JSON. Perlu di-await juga karena prosesnya asinkron. Hasilnya adalah objek JS yang bisa diakses langsung.
try / catchTangkap error yang mungkin terjadi — network error, server down, JSON rusak. Tanpa ini, error tidak tertangani dan program bisa crash tanpa pesan yang jelas.

Tips & Trik JavaScript

Shortcut dan kebiasaan coding JS yang bikin kode lebih bersih dan efisien.

tips.js
// Optional chaining — aman akses properti nested
const kota = user?.address?.city;
// Tanpa ?. bisa error kalau user atau address undefined

// Array destructuring
const [pertama, kedua] = [10, 20, 30];
console.log(pertama);  // 10

// Konversi tipe
Number("42");           // 42
String(42);            // "42"
parseInt("42px");      // 42 (ambil angka di awal)
parseFloat("3.14");   // 3.14

// String methods yang sering dipakai
"halo dunia".toUpperCase();        // "HALO DUNIA"
"  spasi  ".trim();               // "spasi"
"a,b,c".split(",");                // ["a","b","c"]
"halo".includes("hal");           // true
"halo dunia".replace("dunia","JS");  // "halo JS"
ShortcutFungsi
user?.nameOptional chaining — tidak error kalau user undefined
value ?? "default"Fallback kalau null/undefined
[...arr1, ...arr2]Gabung dua array jadi satu
{...obj1, ...obj2}Gabung dua object
Math.max(...arr)Nilai terbesar dalam array
arr.sort((a,b)=>a-b)Urutkan array angka ascending
Jangan pakai var — pakai const dan let. Jangan pakai == — selalu pakai ===. Dua kebiasaan ini saja udah menghilangkan banyak bug umum di JavaScript.
